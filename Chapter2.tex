\chapter{Разработка программного комплекса для численного моделирования}

Для численного исследования математической модели, описанной в предыдущей главе,
был разработан специализированный программный комплекс. Основой для его
реализации послужил язык программирования Python ввиду его развитой
инфраструктуры для научных вычислений \cite{python_sci}.
В частности, для векторных операций
использовалась библиотека NumPy, для построения графического интерфейса --
библиотека визуальных компонентов PyQt5, а для визуализации -- библиотека
Matplotlib \cite{matplotlib}. Численное решение системы обыкновенных
дифференциальных уравнений (ОДУ) выполняется с помощью классического метода
Рунге-Кутты 4-го порядка \cite{yang_runge_kutta}.

\section{Описание численного метода Рунге-Кутты}

Для численного решения систем обыкновенных дифференциальных уравнений (ОДУ),
особенно нелинейных, для которых аналитическое решение не существует, широко
применяются методы Рунге-Кутты. В данной работе для моделирования динамики
опухолевых процессов используется классический метод Рунге-Кутты 4-го порядка,
реализованный на языке Python.

Ниже представлена общая форма системы ОДУ первого порядка \ref{eq:general_ode}:
\begin{equation}
  \label{eq:general_ode}
  \frac{dy}{dt} = f(t, y),
\end{equation}
где $y$ -- вектор зависимых переменных (в нашем случае $[T, I, C]$), $t$ --
независимая переменная (время), и $f(t, y)$ -- вектор-функция, описывающая
правые части системы.

Основное итерационное соотношение метода Рунге-Кутты 4-го порядка
\ref{eq:rk4_update}, позволяющее вычислить значение на следующем шаге, имеет вид:
\begin{equation}
  \label{eq:rk4_update}
  y_{n+1} = y_n + \frac{1}{6}(k_1 + 2k_2 + 2k_3 + k_4),
\end{equation}
где $k_1, k_2, k_3, k_4$ представляют собой взвешенные оценки
\ref{eq:rk4_coefficients} наклона функции $f(t, y)$ в различных точках
внутри интервала $[t_n, t_n + h]$:

\begin{equation}
  \label{eq:rk4_coefficients}
  \begin{cases}
    k_1 &= h \cdot f(t_n, y_n) \\
    k_2 &= h \cdot f(t_n + \frac{1}{2}h, y_n + \frac{1}{2}k_1) \\
    k_3 &= h \cdot f(t_n + \frac{1}{2}h, y_n + \frac{1}{2}k_2) \\
    k_4 &= h \cdot f(t_n + h, y_n + k_3).
  \end{cases}
\end{equation}

Данный метод имеет четвертый порядок точности, что обеспечивает хороший баланс
между точностью и вычислительной сложностью для решения поставленной задачи
\cite{butcher2016numerical}.

Процесс решения системы начинается с известных начальных условий $y_0$ в момент
времени $t_0$. На каждом шаге итерации, алгоритм последовательно вычисляет четыре
вспомогательных коэффициента ($k_1, k_2, k_3, k_4$), каждый из которых
представляет собой оценку производной в определенной точке внутри шага $h$. Затем
эти коэффициенты взвешенно суммируются для получения наиболее точного приращения,
которое добавляется к значению на предыдущем шаге, $y_n$, для нахождения
следующего значения, $y_{n+1}$. Этот процесс повторяется до тех пор, пока не
будет достигнут конечный момент времени моделирования.

\section{Проектирование интерфейса и логики работы программного комплекса}

Созданный программный комплекс для моделирования динамики опухолей имеет
определенную структуру. Далее будет описан его пользовательский интерфейс,
а также архитектуру и алгоритмы, которые для
наглядности представлены в виде диаграмм.. Ниже мы детально рассмотрим как его
внешнюю часть -- пользовательский интерфейс, так и внутреннюю --  архитектуру
и алгоритмы, которые для наглядности представлены в виде диаграмм.

Макет пользовательского интерфейса, предназначенный для управления
моделированием и анализа результатов, показан на рисунке

  \caption{Макет графического пользовательского интерфейса.}

Левая часть окна содержит основной блок визуализации для отображения

Правая часть окна включает три блока. Блок <<Управление терапией>>
позволяет настраивать тип (постоянная, импульсная) и интенсивность
четырех терапевтических воздействий. Блок <<Параметрический анализ>>
служит для выбора варьируемого параметра и запуска серии расчетов.
Результаты параметрического анализа отображаются в третьем блоке --
дополнительной области визуализации.

\subsection{UML-диаграмма последовательности основных сценариев
взаимодействия с программным комплексом}

Для детального описания взаимодействия ключевых частей системы во времени
используется диаграмма последовательности (UML Sequence Diagram)
\cite{Fowler2004UML}, представленная на рисунке \ref{fig:uml_sequence}.
Она иллюстрирует, как объекты обмениваются
сообщениями для выполнения двух основных сценариев работы: стандартной
симуляции и параметрического анализа. Участниками взаимодействия являются
три логических объекта: пользователь (внешний актор, инициатор событий),
интерфейс (отвечает за всю визуальную часть и обработку действий) и модель
(инкапсулирует все математические вычисления).

\begin{figure}[ht!]
  \centering
  \includegraphics[width=0.9\textwidth]{fig/uml_sequence.png}
  \caption{Диаграмма последовательности основных сценариев работы.}
  \label{fig:uml_sequence}
\end{figure}

Первый сценарий, <<симуляция>>, инициируется действием пользователя, таким
как <<изменение параметра>> на одном из ползунков. В ответ интерфейс
вызывает собственный внутренний метод $run\_simulation()$, который собирает
актуальные данные с элементов управления. Затем он делегирует вычисление
объекту модель, отправляя синхронное сообщение $solve\_rk4()$. Это означает,
что интерфейс приостанавливает свою работу и ожидает, пока модель
выполнит все итерации численного метода. После завершения расчетов модель
возвращает вычисленные траектории (массивы времени и концентраций)
сообщением-ответом. Получив эти данные, интерфейс вызывает собственный
метод для визуализации, обновляя основной график.

Второй сценарий, <<параметрический анализ>>, запускается по <<запросу
анализа>> от пользователя (нажатию кнопки). Интерфейс также вызывает
внутренний метод выполнение анализа, но дальнейшая логика отличается.
Активируется рамка $loop$ (цикл), которая показывает, что последующие
действия будут многократно повторяться для каждого значения исследуемого
параметра. Внутри этого цикла интерфейс на каждой итерации вызывает
модель для проведения полного расчета ($solve\_rk4()$) и получает в ответ
лишь одно конечное значение. После завершения цикла, когда все конечные
значения собраны, интерфейс выполняет финальное действие -- вызывает
внутренний метод для визуализации итогового графика зависимости на
второй, аналитической, области отрисовки.

\subsection{Алгоритм численного решения системы ОДУ методом Рунге-Кутты 4-го порядка}

Для наглядной демонстрации работы вычислительного ядра программы, реализующего
метод Рунге-Кутты 4-го порядка, была составлена подробная блок-схема
алгоритма. Она представлена на рисунке \ref{fig:rk4_flowchart} и детализирует
каждый шаг, выполняемый функцией $solve\_rk4()$ для получения решения системы
ОДУ.

\begin{figure}[ht!]
  \centering
  \includegraphics[width=0.5\textwidth]{fig/rk4_flowchart.png}
  \caption{Блок-схема алгоритма численного решения методом Рунге-Кутты 4-го порядка.}
  \label{fig:rk4_flowchart}
\end{figure}

Процесс, отображенный на блок-схеме, начинается с получения на вход
начальных параметров: вектора начальных условий $y_0$, начального и конечного
моментов времени $t_0$ и $t_{final}$, шага интегрирования $h$ и самой функции
правых частей системы $f(t, y)$. Далее следует этап инициализации, на котором
создаются массивы для хранения результатов, а текущему времени и состоянию
системы присваиваются начальные значения.

Основная часть алгоритма -- это цикл, который выполняется, пока текущее
время $t$ меньше конечного времени $t_{final}$. Внутри цикла последовательно
производится расчет четырех вспомогательных коэффициентов $k_1, k_2, k_3, k_4$
в соответствии с формулами \eqref{eq:rk4_coefficients}. Затем на их основе
вычисляется новое состояние системы $y_{new}$ по формуле \eqref{eq:rk4_update}.
В конце итерации текущее время увеличивается на
величину шага $h$, а новое состояние $(t, y_{new})$ сохраняется в массивы
результатов. После выхода из цикла, когда моделирование завершено, функция
возвращает массивы с полной траекторией решения.
